# 收藏界面卡牌文本投影设计

## 背景

当前投影只基于卡牌 `richText` 和部分 tags 生成 `displayText`，并用局部替换生成 `text`。这会遗留固定字符串键、未填充的引用参数和未转换的游戏控制标记，不能稳定表示收藏界面中的卡牌文本。

本设计的目标是使静态收藏界面范围内：

- `displayText` 等于传入游戏 UberText 的最终格式化文本内容。
- `text` 等于该内容按 UberText 纯文本规则移除样式后的文本，并保留一项有意的无空格书写语言换行偏移。

运行时实体状态、法术强度、区域、控制者、战场加成和控件宽度导致的自动折行不属于本设计范围。

## 范围与边界

投影处理静态 `CardTextBuilder` 行为：分段选择、静态 tag 参数、固定字符串、跨卡引用名称和收藏界面可确定的文本控制标记。

无法仅凭卡牌定义确定输出的 builder、缺少引用卡牌本地化名称、缺少固定字符串键或出现未支持的控制标记时，投影必须失败。不得保留键名、占位符或原始模板作为降级结果。

`herald` 是固定字符串的例外选择规则：卡牌定义恰好包含一个受支持职业时，使用该职业对应的 `GAMEPLAY_HERALD_*` 名称；中立、多职业或不支持的职业使用 `GAMEPLAY_HERALD_DEFAULT`。这复刻游戏 EntityDef 路径在无法由定义唯一确定职业时的默认回退，不依赖玩家当前收藏筛选职业。

`displayText` 不包含 UberText 根据控件宽度、字体和字号动态插入的自动折行；这些结果不具有静态唯一值。

## 固定字符串

固定字符串的上游来源为 hsdata 仓库的 `Strings/<locale>/*.txt`。该来源是按语言组织的制表符分隔键值文件，并会持续新增或修改内容。

本轮不读取该目录，也不新增数据库表。项目在 TypeScript 源码中维护一份只读的最新字符串常量，覆盖当前领域模型支持的 14 种语言和所有静态 builder 所需键。更新该常量时以 hsdata `Strings` 为对照。

固定字符串目录必须按语言和键精确查找。任一查找失败都抛出包含卡牌、builder、语言和键的错误。

## 投影流程

每张卡牌在同一静态投影上下文中解析，包含当前卡 tags、同版本卡牌的 dbfId 索引、同语言卡名索引和固定字符串目录。

处理顺序必须与游戏文本路径一致：

```text
richText
-> CardTextBuilder 静态解析
-> TextUtils 静态数值与控制标记转换
-> GameStrings 语言规则
-> displayText
-> text
```

所有已声明的 `textBuilderType` 都必须有显式的静态收藏界面行为。builder 可以选择文本段、替换参数、解析引用卡名和固定字符串，或在游戏定义为空时返回空文本；不得使用“原样返回模板”代表未实现。

语言规则仅在所有 builder 和固定字符串替换完成后执行一次。韩语规则需复刻游戏对韩文、拉丁字符、数字及标记尾字符的前置字符判定。

## displayText

`displayText` 保留游戏的可见格式化表达，例如 `<b>` 与显式换行。它完成静态 builder、固定字符串、参数、数值和语言规则处理，但不执行控件宽度相关的自动换行。

文本控制标记的处理遵循游戏 `TextUtils.TransformCardText` 和 UberText 的渲染前规则。无法静态解析的控制标记是投影错误。

## text

`text` 从最终 `displayText` 派生。基础规则严格移植 UberText 的 `RemoveMarkupAndCollapseWhitespaces`：移除 `<...>` 富文本与 `[b]`、`[d]`、`[x]` 控制标记；将空格、制表符、回车、换行和 `_` 视为同类空白并压缩；处理连字符后换行的连接规则；最后执行 `Trim()`。

`<_` 和 `_>` 需按 UberText 的尖括号转义规则先还原为字面尖括号与相邻空格，不能误作为富文本标签删除。

在调用上述 UberText 纯文本规则前，对 `zhs`、`zht`、`ja`、`th` 删除显式的 `CR` 和 `LF`。这是有意偏离游戏搜索规则：这些无空格书写语言可能被卡面折行切断，删除折行可避免搜索文本被插入空格。其他语言的换行完全遵循 UberText 的空白压缩规则。

实现处必须保留一条简短英文注释说明这一有意偏移及其搜索目的。

## 测试与验证

新增聚焦测试覆盖：

- 固定字符串解析、跨卡名称解析、静态 builder 分段和参数替换。
- 静态数值和格式控制标记，以及缺键、缺引用和未知控制标记的失败路径。
- 韩语语言规则的韩文、拉丁字符、数字和标记尾字符样例。
- UberText 纯文本规则的富文本、方括号控制标记、下划线、转义尖括号、连字符换行及空白压缩。
- `zhs`、`zht`、`ja`、`th` 的换行偏移，以及其他语言保持 UberText 默认行为。

测试使用真实 hsdata 文本样例作为 fixture，但不依赖 Unity UI、字体或动态自动换行。
