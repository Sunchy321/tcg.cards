# Hearthstone Set 管理页评审

## 结论

设计可实施，进入实现阶段。

## 评审要点

- 页面范围聚焦于 `set` 与 `set_localizations`，边界清晰
- 复用现有控制台 `Tag` 页交互模式，能够降低实现复杂度
- ORPC 接口仅需 `list / get / update`，足以支撑当前管理需求
- `update` 使用事务重写 localization，可以保证数据一致性

## 风险

- `dbfId` 修改会影响依赖映射的后台逻辑
- `releaseDate` 当前仍是文本字段，需要允许占位空字符串继续存在

## 建议

- 列表页优先直接在数据库中过滤，避免全量加载后在内存中过滤
- 页面文案中提示 `dbfId` 与导出映射相关，避免误操作
