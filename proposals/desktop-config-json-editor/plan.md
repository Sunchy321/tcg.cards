## 目标

在保留现有表单设置的前提下，为 desktop 补齐：

- `desktop-config.json` 原始编辑器
- 配置文件路径与入口动作

并保证：

- 原始编辑不会丢失未知字段
- 原始编辑与表单设置共用同一个底层配置源
- 页面文案能明确区分配置文件与 keyring

## 实施原则

- 原始编辑器是高级入口，不替代表单设置
- 配置文件读写走整文件模型，而不是拆成字段回写
- key 提示首轮只做静态已知 key 提示
- 入口动作优先做“打开配置文件夹”和“复制路径”

## 实施步骤

### 1. 补齐 desktop 原始配置文件读写与路径信息命令

至少补齐：

- `desktop_get_raw_config`
- `desktop_set_raw_config`
- `desktop_get_config_file_info`

返回信息至少覆盖：

- 原始 JSON 文本
- 配置文件绝对路径
- 配置目录绝对路径
- 文件是否存在

### 2. 调整数据库设置页入口与独立原始配置页面骨架

页面至少包含：

- 数据库设置页标题右侧的编辑器入口
- 数据库设置页标题右侧的打开配置文件夹按钮
- 独立原始配置页面
- 当前文件路径
- 复制路径按钮
- 重载按钮
- 保存按钮
- 编辑器容器

### 3. 接入 Monaco JSON 编辑器基础能力

首轮至少支持：

- JSON 语法高亮
- 编辑脏状态
- 保存前解析错误提示
- 重新加载
- 格式化
- 接近 VS Code 的原文编辑体验

### 4. 增加配置文件入口动作

首轮至少支持：

- 打开配置文件夹
- 复制配置文件路径

如果跨平台实现成本可控，可额外支持：

- 用默认编辑器打开配置文件

### 5. 接入基础 key 提示与最小结构校验

首轮提示范围至少覆盖：

- `version`
- `externalDatabaseConnectionString`
- `games`
- `games.hearthstone`
- `games.hearthstone.hsdata.repoPath`
- `games.hearthstone.publish.publishTargetId`
- `games.hearthstone.publish.environment`
- `games.hearthstone.publish.targetFingerprint`

保存前最小结构校验至少覆盖：

- 顶层必须是对象
- `version` 若存在则必须是数字
- `games` 若存在则必须是对象

### 6. 完成基础验证并更新文档口径

至少验证以下场景：

- 能读取当前配置文件文本
- 能在编辑器中修改并保存合法 JSON
- 非法 JSON 会阻止保存并显示错误
- 未知字段保存后不丢失
- 配置文件夹入口可用
- 路径复制可用
- 页面文案明确说明 keyring 不在当前 JSON 文件中
