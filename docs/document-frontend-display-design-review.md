# `document-frontend-display-design.md` 评审意见

## 1. 评审结论

通过，可以作为实现前的设计基线。

## 2. 总体评价

该细化版方案相比上一轮结论更完整，已经补齐了：

- 页面层信息架构
- View Model 草案
- 路由设计
- BFF / API 草案
- 交互层行为
- 分阶段落地顺序

它已经不是“方向性建议”，而是一份可进入编码拆解的前端方案。

## 3. 通过项

### 3.1 将前端能力与底层模型清晰脱钩

方案明确要求前端消费聚合后的页面模型，而不是直接消费：

- `DocumentNode`
- `DocumentNodeChange`
- `DocumentChangeReview`

这点非常关键。否则前端会承接过多领域规则，后续迭代成本会显著上升。

### 3.2 `nodeId` 与 `entityId` 的职责边界清晰

方案明确规定：

- 阅读页按 `nodeId`
- 历史页按 `entityId`

这个边界是正确且必要的，能够避免未来 rename / move 场景的历史断裂问题。

### 3.3 对比页从文本 diff 升级为结构化 change card

方案没有继续沿用旧 `rule/diff` 的纯双栏模式，而是为：

- `added`
- `removed`
- `modified`
- `moved`
- `renamed`
- `split`
- `merged`

设计了不同层级的展现策略。这与底层 change 模型是一致的。

### 3.4 公共站点与控制台职责清晰

方案明确区分：

- `site-magic` 只展示稳定且面向读者的信息
- `site-console` 承担导入、排查、审核、修正

这能有效避免公共站点后台化，也降低误暴露未审核结果的风险。

### 3.5 入口路径更直接

方案现已改为：

- 用户访问 `/document/[slug]` 时直接进入最新版阅读
- 历史版本通过 `version` query 切换

这个入口更符合阅读场景，也减少了“先到首页再点击一次”的无效路径。

## 4. 实施中需要保持的约束

### 4.1 公共站点必须默认使用 `reviewed`

这是本方案最重要的产品约束之一。若公共站点直接展示 `snapshot`：

- 用户会看到待审核结论
- 审核后页面语义会发生反转
- 搜索引擎可能缓存不稳定内容

因此：

- 公共站点默认走 `reviewed`
- 控制台默认走 `snapshot`
- 仅在诊断场景下允许人工切换

### 4.2 第一版右栏上下文不要过重

阅读页右栏非常容易失控，实施中建议第一版只保留：

- 当前 `nodeId`
- 历史入口
- 最近一次变更摘要

不要一次性接入：

- 审核状态
- 关系图
- 调试字段

### 4.3 最新版解析规则要稳定

既然 `/document/[slug]` 直接承接最新版阅读，实现时应明确：

- 默认取当前 `active` 且最新的版本
- 当 `version` query 存在时优先使用显式版本
- 当指定版本不存在时返回明确的 404 或回退策略

### 4.4 第一版不要过早做复杂图形关系编辑

`split` / `merged` 的关系编辑可以先采用：

- 结构化表单
- 明确的 from / to 列表

而不是直接做图形拖拽编辑器。否则开发量和维护成本会快速膨胀。

### 4.5 需尽早冻结公共 View Model

若没有先冻结：

- `DocumentReaderPage`
- `DocumentComparePage`
- `DocumentEntityHistoryPage`
- `DocumentReviewDetail`

后续页面与后端会反复返工。

## 5. 建议补充到实施阶段的细节

### 5.1 组件复用边界

建议在实现阶段明确：

- 哪些组件属于公共 UI 组件
- 哪些组件只属于 `site-magic`
- 哪些组件只属于 `site-console`

避免后续组件目录混乱。

### 5.2 i18n 文案策略

方案已包含 `localeState`，实施时应同步规划：

- 公共站点展示文案
- 控制台审核文案
- 变更类型翻译

避免后续大量补文案。

### 5.3 SSR 与客户端状态职责

建议在实现阶段明确：

- 阅读页优先 SSR
- diff / history 可按交互复杂度决定 SSR 或 CSR
- 控制台审核页优先 CSR + 明确缓存失效

## 6. 评审结论说明

该方案结构完整、边界清晰、风险控制合理，已达到“可以进入实施拆解”的质量标准。

结论：**评审通过，进入计划阶段。**
